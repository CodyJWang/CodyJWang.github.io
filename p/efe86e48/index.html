¶建站过程

由于编写不及时，导致配置评论系统以及之前的部分并不完善。

为什么要把建站过程也写出来呢，这不就把我的底裤给暴露出来了吗？
其实我是写给自己看的，因为随着网站功能越来越多，我渐渐忘了我究竟都干了些什么，维护起来很不方便。

¶选择博客框架

目前主要有两款很好用的博客框架，WordPress 和 Hexo。简单来说，WordPress 功能强大、易于维护，Hexo 是静态的、维护成本高，这么看来，WordPress 貌似更好。但是，搭建网站需要服务器呀，作为学生党的我哪里买得起！本着能白嫖就白嫖的原则，我选择了 Hexo + Github Pages 这一经典组合。
后来，我崩溃了，这也太难维护了吧！

¶搭建网站雏形

这部分呢，难不倒作为高级 CV 工程师的我，只需发动我的看家本领 Ctrl + C Ctrl + V 便可搞定，轻轻松松(～￣▽￣)～
最终我选择了 Butterfly 博客主题和 WebStack 导航主题。这部分的详细过程我就懒得写了。

¶实现数学公式、流程图的渲染

常见的渲染器有：
hexo-renderer-marked
hexo-renderer-kramed
hexo-renderer-pandoc
hexo-renderer-markdown-it
hexo-renderer-markdown-it-plus

本网站最终选择了 hexo-renderer-pandoc

数学公式渲染引擎主要有MathJax\rm{MathJax}MathJax 和 \rm{K\kern-.2em\raise.21em{\scriptstyle{A}}\kern-.17em T\kern-.14em\lower.5ex{E}\kern-.115em X} 两类。

Butterfly 主题推荐 \rm{K\kern-.2em\raise.21em{\scriptstyle{A}}\kern-.17em T\kern-.14em\lower.5ex{E}\kern-.115em X} ，我也就无脑选择了它，但是随之而来的是其使用的 Markdown 渲染插件 hexo-renderer-markdown-it 带来的一些问题。然后我决定换用其它渲染插件，一并把数学公式渲染引擎也换成MathJax\rm{MathJax}MathJax 。

MathJax\text{MathJax}MathJax 唯一的缺点是渲染慢，但想想我的文章中不会出现太多公式，所以也就无所谓了。

这个很好配置，Butterfly 内置了 Mermaid 图表工具，只需把开关打开即可。

¶配置搜索引擎

一开始是通过 hexo-algoliasearch 插件使用了 Algolia 的搜索服务。等 DocSearch 的申请通过后，换成了 DocSearch。DocSearch 其实也是 Algolia 家的。

在这期间，我被 Algolia 推荐了 GitGuardian，正在试用。

¶配置评论系统

这部分真是花费了很大心思，也正是因此才写这篇建站过程的。

一开始我选择了 Giscus，因为这个的配置最简单，但是使用体验并不好。后来我又选择了 Waline，挺好看的，用到了 Vercel 和 LeanCloud(国际版) 服务，但是很可惜被墙了，国内网络访问不了。

最后我选择了 Twikoo，但是对于部署平台仍有多种选择，如 Netlify、Zeabur 等。我用 GitHub 注册了 Netlify，也进行身份验证了，但是登录不上去，然后换用 QQ 邮箱注册，竟然连注册都不行，就很迷惑。百般无奈，只好选用 Zeabur 了。这玩意儿挺强大的，但是免费版只有 7 天时限，到期前需要你手动延期，每次还只能延到 7 天，真™折磨人！还用了 MongoDB 的免费数据库，这个就挺好的。

评论系统的消息推送服务我选的是 Qmsg，网页端能收到，而 QQ 机器人却无推送，弃了。然后选了 Server 酱，微信端的，但是由于腾讯的不支持，在电脑端不好用，弃了。最后选了 pushplus(推送加)。

反垃圾服务选的是 Akismet，免费版的限制是：网站应具有非商业性质，其中一点是不得插入广告。这，我还想赚点小钱来着。
